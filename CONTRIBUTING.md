# 贡献指南

感谢你愿意让这个项目变得更好。为了让你的建议被快速理解和采纳，请花一分钟读完本文件。

## 怎么提建议

请在 GitHub 上提 Issue，选择对应的模板填写：

- **GitHub Issues**：https://github.com/czl-dev-pub/math-atlas/issues

| 类型 | 模板 | 适合 |
|---|---|---|
| 🐛 知识点 / 事实纠错 | 知识点 / 事实纠错 | 定义写错、教材作者/版次有误、前置依赖关系不对、MSC 编码错误 |
| 📚 教材 / 资源推荐 | 教材 / 资源推荐 | 推荐更好的教材、补充某节点的学习资源 |
| 💡 功能建议 | 功能建议 | 新交互、新功能、可视化改进 |
| 🎯 体验问题 | 体验问题 | 操作不顺手、看不懂、显示异常 |

（两平台的模板内容一致：GitLab 用 `.gitlab/issue_templates/`，GitHub 用 `.github/ISSUE_TEMPLATE/`。）

**请尽量用模板的结构化字段填写**，而不是写成一段散文——这能让我们在几分钟内核对并回应。

## 采纳标准

为了保持内容的可靠性，不同类型的建议有不同的门槛：

### ✅ 容易被采纳
- **事实纠错**：**必须附可核对的来源链接**（教材 ISBN / 出版社页面 / 维基百科 / 豆瓣读书等）。无来源的主张会被搁置或关闭。来源越权威（出版社官网 > 豆瓣 > 个人博客），采纳越快。
- **教材推荐**：注明书名、作者、出版社、版次（如有 ISBN 更好），便于核对。
- **明确的 bug**（显示错乱、点击无反应等）：附截图 + 复现步骤。

### ⚠️ 谨慎处理（不一定采纳）
- **主观学习顺序建议**（"我觉得应该先学 X 再学 Y"）：除非有权威课程大纲支撑，否则默认不改动——学习路径带有一定主观性，频繁调整会破坏图谱的稳定性。
- **大规模重构建议**：需要评估工作量和兼容性。

### ❌ 不在范围内
- 增加全新数学分支节点（当前聚焦 86 节点的质量，而非扩张）。
- 与数学学习导航无关的需求。

## 处理流程

1. 你提 Issue（用模板，GitLab 或 GitHub 任一）。
2. 我们核对来源 / 复现问题。
3. 通过 **Merge Request**（GitLab）/ **Pull Request**（GitHub）提交修改，描述里写清"改了什么、依据什么"。
4. 合并到主分支，Issue 关闭。

> 即便你不写代码，只提 Issue 也非常有价值——核对与实现由维护者完成。

## 直接提 Merge Request / Pull Request

如果你希望自己改代码：

1. Fork 本仓库。
2. 改动 `math-atlas.html`（纯前端单文件，浏览器打开即可本地预览）。
3. 提 PR，说明改动内容与依据。
4. 等待 review。**事实性改动同样需要在 PR 描述里附来源。**

## 代码 / 内容规范

- `math-atlas.html` 是单文件应用，所有数据（`NODES` 对象）内联其中。改节点数据时保持已有字段的顺序，避免破坏本地存储的知识点勾选状态（详见 [开发者文档](docs/DEVELOPMENT.md)）。
- 修改后请用浏览器实际打开验证渲染，不要只看代码。
