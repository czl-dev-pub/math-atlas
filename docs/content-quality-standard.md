# 数学导航地图 · 节点内容质量标准（spec）

## 1. 背景与问题

当前 86 个节点的内容质量存在明显梯度：
- **基础层（18 节点）**：有人教版课标背书，含 items 知识点清单，质量较高、可溯源。
- **大学核心层（约 40 节点）**：def/why 为人工概括，缺 items，资源只有书名。
- **大学专业层（约 28 节点）**：def/why 为人工主观概括，无 items、无审核，资源未验证，可能存在事实错误。

**已发现的具体问题**（待修正）：
1. 节点 `homology_alg` 的资源写"《同调代数导论（佟文廷）》"，但权威推荐是陈志杰《同调代数》或 Weibel/Rotman——作者可能记错。
2. 大学层 60+ 节点全部缺少 items 知识点清单，与基础层细节程度不一致。
3. 所有节点的 def/why/prereq/depth 均为人工编写，无权威来源校验。
4. 资源（书名/作者/课程）从未核对真实性与版本准确性。

## 2. 目标

对全部 86 个节点执行**全面溯源重做**（用户已确认采用方案 A，最严格）：
- 每个需要事实性判断的字段都对照权威来源核对、修正。
- 全部节点补齐 items 知识点清单，大学层基于公认教材整理并标注依据。
- 全部资源联网核对书名/作者/版本/可得性。
- 每个字段标注来源；无法溯源的不得伪造可信，必须如实标注存疑。

定位原则（用户确认）："严格溯源" + "透明分级"结合——能溯源的必须溯源并标记权威，溯源不了的诚实标注待校对/存疑，让用户知情。

## 3. 各字段的审核标准

### 3.1 `name`（节点名）
- 标准：使用该分支在中文数学界的通行名称。
- 溯源依据：人教版课标（基础层）/ MSC2020 中译 / 维基百科中文 / 公认教材。
- 合格判据：名称与上述至少一个来源一致。

### 3.2 `def`（一句话定义）
- 标准：准确、无歧义、不超过 40 字，符合该分支的学术定义。
- 溯源依据：维基百科（中/英）/ 教材绪论 / 数学百科（如 Wolfram MathWorld / Encyclopedia of Mathematics）。
- 合格判据：定义的核心术语与权威来源一致；不得有事实错误。

### 3.3 `why`（它解决什么问题）
- 标准：说明该分支的实际动机与应用，成人化、目标导向。
- 溯源依据：教材前言/绪论、应用领域的权威介绍。
- 合格判据：提到的应用领域（如"机器学习用到矩阵"）必须有据可查；不得编造应用联系。

### 3.4 `prereq`（前置依赖）
- 标准：列出学习该分支前必须掌握的节点，关系正确。
- 溯源依据：权威教材的预备知识章节 / 大学课程大纲的 prerequisite。
- 合格判据：每条 prereq 能在至少一个权威来源的"预备知识"中找到对应；不应有错误的前置（如把不需要的列为前置）。

### 3.5 `msc`（MSC2020 编码）
- 标准：符合 MSC2020 官方分类。
- 溯源依据：MSC2020 官方 CSV（已下载校准过）。
- 合格判据：编码存在于官方分类，且语义对应（已基本完成，复核即可）。

### 3.6 `items`（知识点清单）
- 标准：列出该分支涵盖的核心知识点，粒度适中（每个节点 8-20 个）。
- 溯源依据：
  - 基础层：人教版教材目录（已做，复核）。
  - 大学层：**公认经典教材的目录**——在节点级别标注 items 的整体依据（如 `items_src:'同济《高等数学》第七版目录'`），而非每条知识点单独标章节（避免冗余）。
- 合格判据：items 中的每个知识点都对应教材的某个章节/小节；不得凭空编造知识点。

### 3.7 `res`（推荐资源）
- 标准：书名、作者、版本**全部核对真实存在且准确**；课程核对仍在提供。
- 溯源依据：searxng 搜索 + 出版社/官网/豆瓣读书/出版社目录核实。
- 合格判据：每条资源经搜索确认（书名、作者、ISBN 或出版社可查）；查不到的标 `[未验证]` 并从主推荐降级到"参考"。

### 3.8 `depth`（学到什么程度算够）
- 标准：对每个目标给出具体的掌握程度描述。
- 溯源依据：该目标领域的权威课程要求 / 行业共识。
- 合格判据：描述的掌握程度（如"必须熟练"）有依据，不武断；无依据的标注"参考性建议"。

## 4. 质量等级标记（qa 字段）

每个节点增加 `qa` 字段，标记整体质量等级，让用户知情：

| 等级 | 含义 | 判据 |
|------|------|------|
| **A** | 权威背书 | 所有字段溯源完成，且核心依据来自官方课标/MSC/公认经典教材 |
| **B** | 已核对 | 所有字段经搜索核对，依据来自维基/教材但非"官方课标" |
| **C** | 待校对 | 部分字段已核对，部分仍为人工判断未溯源 |
| **D** | 存疑 | 存在已知问题或无法溯源，需用户当心 |

UI 上节点详情面板显示 qa 等级（如徽章 `A 权威`/`C 待校对`），全景图可选按 qa 着色。

## 5. 执行方式（用户确认：一次性全做）

### 5.1 工具链
- 搜索：searxng（`https://searxng.redcoast.dpdns.org/search?q=...&format=json`），通过 curl + node 解析调用。
- 网页内容：webReader（读维基/教材页面细核）。
- 修改：直接 Edit math-atlas.html 的节点数据。

### 5.2 流程（每个节点）
1. searxng 搜该分支的权威定义/教材/课程。
2. webReader 读最权威的来源页（维基中英/教材目录）。
3. 逐字段核对修正 def/why/prereq/msc。
4. 基于教材目录整理 items 并标注依据。
5. 核对 res 真实性（书名/作者/版本/可得性）。
6. 打 qa 等级。
7. 验证（jsdom 数据完整性 + 真实浏览器抽样截图）。

### 5.3 工作量预估（诚实说明）
- 86 节点 × 平均每节点 2-3 次搜索 + 1-2 次网页阅读 ≈ 200-300 次搜索调用。
- 这是一个**很长的过程**，会分多次工具调用完成。
- 基础层（18）已基本达标，主要工作量在大学层（68 节点）。

## 6. 风险与诚实说明
1. **大学层 items 的局限**：大学数学无统一课标，items 基于"公认教材"整理，不同教材的章节划分有差异，无法像人教版那样唯一权威——这部分质量会注明"基于XX教材"，诚实呈现。
2. **资源版本变化**：教材可能再版、课程可能下线——核对以"当前可查"为准，标注核对日期。
3. **冷门节点**：部分冷门分支（如纽结理论、信息几何）的中文资源稀少，可能只能溯源英文资料——会如实标注。
4. **我不会的事**：无法保证 100% 无错，但会保证每一处都有溯源依据、无法溯源的诚实标注，绝不伪造可信。

## 7. 不做的事（YAGNI）
- 不重写已工作的交互层（聚焦内容质量）。
- 不增加新节点（聚焦现有 86 个的质量）。
- 不改变图谱布局/交互（那是后续意见2/4的事）。
- 本 spec 只管"内容质量标准与执行"，学习入口（意见4）、问卷报告（想法）是独立后续工作。

## 8. 验收标准
- 全部 86 节点都有 items 字段且标注依据。
- 全部 res 经核对，书名/作者可查。
- 全部 def/why/prereq 经至少一个权威来源核对。
- 全部节点有 qa 等级标记。
- jsdom 验证无悬空引用、字段完整。
- 抽样真实浏览器截图确认 qa 徽章显示正常。
